# Rezon — статус проекта (хендофф между сессиями)

> Прочитай это ПЕРВЫМ при продолжении работы. Затем при необходимости: `BRIEF.md` (ТЗ), `PLAN.md` (слои), `STRATEGY.md` + `COMPETITOR-CONTENT.md` (конкуренты/позиционирование), `ROLEPLAY-DESIGN.md` (движок роль-плея), `supabase-schema.sql`.

## Что это
**Rezon** — Telegram Mini App для голоса/речи/уверенности (freemium, RU/СНГ). Бренд воронки/бота — «Голос на миллион» (Алмат). Запуск: тёплый 1 авг 2026, публичный 15 авг 2026.

## Стек и где что
- Репозиторий: `github.com/sultumovv-lab/golos-app`. Фронт — один файл `index.html` (vanilla JS). API — Vercel-функции: `api/chat.js` (AI-коуч), `api/plan.js` (онбординг-план), `api/roleplay.js` (роль-плей: converse/feedback/build/coach + мульти-персона), `api/state.js` (синк прогресса), `api/voice.js` (голосовой дневник).
- Фронт на **Vercel** (`golos-app.vercel.app`). Бот `bot.py` на **Railway** (`@kaifgolos_bot`).
- **Vercel env:** `ANTHROPIC_API_KEY`, `BOT_TOKEN`, `SUPABASE_URL`, `SUPABASE_SERVICE_KEY`.
- **Supabase:** проект `ksvnxphthvflzmjgsdzm`, таблицы `users` + `voices`, приватный бакет `voices`.

## Готово (всё на проде)
- Бот: лист ожидания, админ-рассылка, приложение открыто всем (меню + кнопка в /start).
- Приложение (слой 1): дизайн «Тёплая студия», экран «Сегодня» + ежедневный цикл, онбординг-тест + AI-план, геймификация (XP/уровни/стрик/энергия/ачивки), freemium + paywall (триал 3 дня, $7/мес).
- Движок роль-плея: персонажи с лицами + карточка исхода/скоринга; мульти-персонаж «совещание»; «Твоя ситуация» (генерация под реальный разговор); **4 режима** (persuade/difficult/connect/support — поведение и метрики адаптируются); Алмат-коуч (💡 совет по ходу); библиотека по категориям.
- Слой 2 (Supabase): синк прогресса (`/api/state`) + голосовой дневник (`/api/voice`: запись «до/после» → Storage, прослушивание, доступ владельца). Личность проверяется по Telegram initData (HMAC c BOT_TOKEN).
- Полировка: гибрид-иконки (инлайн-SVG для нав/контролов, эмодзи для персонажей/категорий), анимации (плей-эквалайзер, активная иконка нав-бара).

## Операционные заметки (важно!)
- **Деплой = git push в `main`** → Vercel сам пересобирает. Бот: push → Railway сам редеплоит.
- **GitHub-пуш:** у ассистента нет постоянного доступа. Каждый раз владелец создаёт свежий PAT (`repo` scope) на github.com/settings/tokens, присылает, ассистент пушит, токен можно удалить.
- **Локальный предпросмотр** (python http.server) НЕ выполняет `/api/*`. AI/бэкенд проверять curl'ом на проде или вживую в Telegram (там приходит подписанный initData).
- Запись голоса — в формате устройства (iOS = mp4/aac), иначе iOS не проигрывает.
- Превью-сервис: `.claude/launch.json` сервис `rezon` (порт 4173).

## Как продолжить в новой сессии
Память ассистента (факты проекта) сохранена в авто-памяти и подхватывается сама, если запускать `claude` из той же папки. Плюс этот файл — источник правды. Первая фраза в новой сессии: «Продолжаем Rezon — прочитай docs/STATUS.md и свою память, дальше делаем контроль AI-расходов».
