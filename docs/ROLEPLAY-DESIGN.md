# Rezon — движок ролевых игр (утверждённая концепция)

**Тезис:** Rezon = «спарринг реальных, в т.ч. эмоциональных разговоров» с живым AI и реальным коучем Алматом. Не «инструмент порепетировать» (Yoodli) и не «качалка голоса» (Vocal Image) — а личный коуч, который бросает в реальные разговоры и вытаскивает из них.

**Решение по аватарам:** оживлять НЕ нужно. Персонаж = статичное фото/портрет + имя + роль (как делает сам Yoodli). Голос — браузерный TTS сейчас, ElevenLabs (Алмат) позже.

## Модель данных
- **Персонаж (persona):** `{ id, name, role, photo, agenda, recurring }`. `agenda` — мотив/характер для системного промпта. `photo` пока emoji-портрет/инициалы, позже — сгенерированные портреты.
- **Сценарий:** `{ id, title, category, situation, personas:[id], difficulty, opener }`. Один персонаж = диалог, 2–4 = «совещание».
- **Категории:** Карьера, Общение, Конфликт, Свидания/Личное, Публичные выступления.

## 6 механик (всё утверждено)
1. **Одиночный роль-плей** (есть) → добавить **лицо персонажа** + **исход/скор**.
2. **Мульти-персонаж «совещание»** (флагман Yoodli): сетка 2–4 фото, активный спикер подсвечен + имя + субтитры. Claude тегирует, кто из персонажей отвечает.
3. **«Твоя ситуация» — конструктор голосом:** юзер описывает свой реальный разговор → Claude генерит персонажа+сценарий → спарринг под его задачу. (`/api/roleplay` phase `build`)
4. **Повторяющиеся персонажи** («спарринг-партнёры»): «Виктор — жёсткий клиент», «вечно недовольный начальник». История встреч (localStorage в v1, сервер позже).
5. **Исход + скоринг:** в конце Claude возвращает результат («Сделка закрыта / Совет сомневается / Тебя не взяли») + баллы (ясность / уверенность / убедительность). Карточка результата. (`/api/roleplay` phase `feedback` → структурный JSON)
6. **Алмат-коуч в углу ринга:** кнопка «💡 совет Алмата» в звонке → быстрая подсказка по ходу диалога. (`/api/roleplay` phase `coach`)

## Стартовый набор сценариев (конкретика)
- **Карьера:** Собеседование · Просьба о повышении · Жёсткий клиент (Виктор, recurring) · Питч совету (мульти: CFO+Product+Procurement)
- **Конфликт:** Реакция на пассивную агрессию · Сказать «нет» начальнику · Ответ на резкую критику
- **Общение:** Самопрезентация · Нетворкинг · Сложный разговор с близким
- **Свидания/личное:** Первое свидание · Неловкая пауза · Признание чувств

## Порядок сборки
- **B1 — Фундамент:** модель персонажей + лица в текущих сценариях + **карточка исхода/скоринга**. На этом стоит всё остальное; сразу поднимает «звонок» до уровня Yoodli + добавляет ставки.
- **B2 — Мульти-персонаж «совещание»** (сетка). Главный дифференциатор.
- **B3 — «Твоя ситуация»** (разговорный конструктор сценария).
- **B4 — Повторяющиеся персонажи + Алмат-коуч + расширение таксономии** до полной библиотеки.

## Чем бьём конкурентов (резюме)
Адаптивные персонажи (живой Claude, не скрипт) · единый разбор + исход · реальный Алмат-коуч · Telegram-нативность + пуши · русский/СНГ-контекст · эмоциональные жизненные сценарии (не только карьера) · цена.
